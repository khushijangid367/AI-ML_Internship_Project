# 🚀 AI & Machine Learning Internship Portfolio

## 📌 Overview
Welcome to my Machine Learning and Artificial Intelligence portfolio! This repository contains the complete source code, methodologies, and documentation for the six core projects I developed during my AI/ML internship. 

These projects demonstrate a full-stack approach to Artificial Intelligence, ranging from foundational data preprocessing and predictive modeling to deploying interactive web applications, REST APIs, and real-time computer vision systems.

## 📁 Project Directory

* **[01] Data_Cleaning:** A foundational Jupyter Notebook pipeline for auditing raw datasets, handling missing values, and executing feature pruning to prepare data for machine learning models.
* **[02] Spam_Message_Detection:** A Natural Language Processing (NLP) machine learning model trained to accurately detect, classify, and filter text-based spam.
* **[03] Chatbot (Gemini):** A responsive web application powered by Google's Gemini 1.5 Flash LLM, featuring contextual conversation memory and secure environment variable management. 
* **[04] Spam_Detection_API:** A robust server architecture wrapping multiple predictive ML models into fully functional REST APIs for seamless frontend consumption.
* **[05] Face_Detection:** A low-latency computer vision pipeline that processes live webcam video streams to identify and track human facial structures using Haar Cascades.
* **[06] Attendance_System:** An advanced facial recognition system utilizing DeepFace (VGG-Face) to verify identities and automatically log timestamped attendance records to a NoSQL cloud database.

## 🛠️ Core Technologies & Tools

* **Languages:** Python, JavaScript
* **AI / Machine Learning:** Google Gemini API, DeepFace, Scikit-learn, Pandas, NumPy , Matplotlib
* **Computer Vision:** OpenCV, Haar Cascades
* **Web & API Development:** React (Vite), Flask, Axios , FastAPI
* **Database & Cloud:** Google Firebase Firestore (Admin SDK)

## 📖 How to Navigate
Each project is contained within its own isolated directory to maintain a clean codebase. 

Please navigate into any individual project folder to view its dedicated `README.md` file. These individual files contain deep dives into the specific technical architecture, logic flows, and step-by-step instructions for running the applications locally on your machine.
